    **TRAIN-01 — Draw value fix:**
    In `Chenapan.get_value_and_terminated()`, find the draw branch that returns `(0, True)` (currently line ~377) and change to `return -1, True`.
    Rationale: Draw = -1 forces aggressive play. With draw = 0, the value head averages toward zero across many draws and losses, collapsing predictions.

    **TRAIN-03 — Deterministic board hashing:**
    1. Add `import hashlib` at the top of the file (after existing imports).
    2. In `Chenapan.get_hash()`, replace the body:
       - BEFORE: `return hash(str(state))`
       - AFTER:
         ```python
         s = np.ascontiguousarray(state)
         return hashlib.md5(s.tobytes()).hexdigest()
         ```
    Rationale: Python's `hash()` is PYTHONHASHSEED-randomized since Python 3.3 — two identical boards hash differently across runs. `np.ascontiguousarray` is required before `tobytes()` because board states may be non-contiguous NumPy views after perspective flips (e.g., after `np.rot90`), which would produce incorrect byte sequences without it.
  </action>
  <verify>
    ```bash
    python -c "import alpha_pan; g = alpha_pan.Chenapan(); import numpy as np; board = g.get_initial_state(); h1 = g.get_hash(board); h2 = g.get_hash(board); print('Hash deterministic:', h1 == h2); print('Hash value:', h1[:8])"
    ```
    Expected: `Hash deterministic: True` — same board produces same hash.

    Also verify draw value by inspection: search `alpha_pan.py` for `return -1, True` — must be present in `get_value_and_terminated`.
  </verify>
  <done>

    - `import hashlib` present at top of file
    - `get_hash()` uses `np.ascontiguousarray(state)` and `hashlib.md5(s.tobytes()).hexdigest()`

  <name>Task 2: Fix TRAIN-02 (temperature sampling) and TRAIN-04 (batch slice off-by-one)</name>

  <action>
    Apply two more targeted fixes to alpha_pan.py:

    **TRAIN-02 — Temperature sampling fix:**
    In `AlphaPan.selfPlay()`, there are two sampling sites — one for `player == 1` and one for `player == -1`. In BOTH branches:
    - BEFORE: `p=np.matrix.flatten(action_probs)` (uses raw probs, ignores temperature)
    - AFTER:
      ```python
      temperature_action_probs = action_probs ** (1.0 / self.args["temperature"])
      temperature_action_probs /= np.sum(temperature_action_probs)  # re-normalize to sum=1
      action_flat = np.random.choice(
          self.game.action_size * self.game.action_size,
          p=np.matrix.flatten(temperature_action_probs)
      )
      ```
    CRITICAL: The `/= np.sum(...)` re-normalization is mandatory. After raising to a power, the array no longer sums to 1 — passing it directly to `np.random.choice` raises `ValueError`. Check that `temperature_action_probs` is re-normalized in EVERY sampling site, not just one.

    **TRAIN-04 — Batch slice off-by-one fix:**
    In `AlphaPan.train()`, find the line with `min(len(memory)-1, ...)` (currently line ~797):
    - BEFORE: `sample = memory[batchIndex:min(len(memory)-1, batchIndex + self.args["batch_size"])]`
    - AFTER: `sample = memory[batchIndex:min(len(memory), batchIndex + self.args["batch_size"])]`
    This ensures the last element is always eligible for inclusion in a batch.
  </action>
